COMPLETED FEATURES

FEATURE 1: Project Setup + File Loading + Preview (DONE)
---------------------------------------------------------
- PySide6 desktop app with QMainWindow
- Browse/Load Excel (.xlsx/.xls) and CSV (.csv/.tsv) files
- Sheet dropdown for multi-sheet Excel files
- Header row spinbox with auto-detect (heuristic: row with most string labels * fill ratio)
- PreviewTableView with DataFrameModel (string cache for fast rendering)
- Preview-only loading (first 1000 rows) for performance
- Background QThread loading for files >50 MB
- pyarrow engine for fast full CSV reads
- Encoding fallback: UTF-8 -> ISO-8859-1
- CSV separator auto-detection (comma, semicolon, tab, pipe)
- Comment line handling (#-prefixed lines)

FEATURE 2: Column Mapping + Export (DONE)
------------------------------------------
- MappingPanel: 16 rows (one per target column from FOCUS_COLS_ETL)
  - Bold label + red asterisk for mandatory columns
  - QComboBox dropdown: "(unmapped)" + all source column names
  - Status indicator: green checkmark, red X, gray dash
- QSplitter: mapping panel (left) + source data preview (right)
- Harmonized Preview table at bottom
- "Apply Mapping" button: runs harmonize_df() on preview data
- "Export..." dropdown menu: CSV, Excel (.xlsx), Parquet
  - Background _ExportWorker for large files
  - Warns if mandatory columns unmapped
  - Default output name: {input_stem}_harmonized.{ext}
- Status bar: "filename | Sheet: X | N rows | Mapped X/16 | Mandatory Y/5"
- Core logic in harmonizer.py:
  - harmonize_df(mapping, source_df) -> output DataFrame
  - export_harmonized(mapping, filepath, output_path, ...) -> row count

FEATURE 3: Save/Load Configuration as JSON (DONE)
---------------------------------------------------
- config_manager.py: MappingConfig dataclass
  - Schema version: "harmonize_config_v1"
  - Fields: name, created, file_settings, column_mappings
  - FileSettings: header_row, sheet_pattern, encoding
  - ColumnMapping: mapping_type (direct/or_fallback/unmapped), source_columns list
- "Save Config" button in Single File tab:
  - Prompts for config name via QInputDialog
  - Saves to JSON with file dialog
  - Includes full fallback info for OR-fallback mappings
- "Load Config" button:
  - Reads JSON, applies header_row setting, restores all mappings + fallbacks
- Round-trip tested: save -> load -> verify all dropdowns restored

JSON format example:
{
  "$schema": "harmonize_config_v1",
  "name": "MCM_Config",
  "created": "2026-02-16T...",
  "file_settings": {"header_row": 2, "sheet_pattern": "Record", "encoding": "utf-8"},
  "column_mappings": {
    "Voltage_V": {"mapping_type": "direct", "source_columns": ["U(V)"]},
    "Current_A": {"mapping_type": "or_fallback", "source_columns": ["Current(A)", "I(A)"]},
    "Power_W": {"mapping_type": "unmapped", "source_columns": []}
  }
}

FEATURE 4: OR-Fallback Mapping (DONE)
---------------------------------------
- Each mapping row has a "+" button to add fallback source columns
- Fallback rows display as "OR [dropdown] [-]" indented under the primary
- _FallbackRow widget: QLabel("OR") + QComboBox + QPushButton("-")
- get_full_mapping() -> {target: [primary, fallback1, fallback2, ...]}
- set_full_mapping() restores primary + all fallback rows
- harmonize_df() with full_mapping: tries each source in order until found in DataFrame
- Fallbacks are saved to JSON config as mapping_type="or_fallback"

FEATURE 5: Batch Processing (DONE)
------------------------------------
- Batch tab UI (batch_tab.py):
  - Input folder picker
  - Output folder picker (auto-set to input/harmonized/)
  - Config JSON file picker
  - File filter field (e.g. "*.xlsx, *.csv")
  - Export format dropdown (CSV, Excel, Parquet)
  - "Skip already processed" checkbox
  - "Scan Files" button -> populates file table
  - "Start Batch" / "Cancel" buttons
  - QProgressBar + progress label
  - QTableWidget: File | Status | Rows | Details
  - Color-coded status: green=success, red=failed, gray=skipped
- Core logic (batch_processor.py):
  - discover_files(folder, filter) -> sorted list of supported files
  - process_single_file(filepath, config, output_folder, format, skip_existing)
    - Auto-detects sheet (uses sheet_pattern from config)
    - Auto-detects header row if config says 0
    - Builds mapping from config (supports fallbacks)
    - Returns BatchFileResult with status/row_count/error
- _BatchWorker runs in QThread, emits file_started/file_done/all_done signals

FEATURE 6: File Browser Tab (DONE)
-------------------------------------
- 3-panel layout: Folder Tree | File List | Quick Preview
- Folder tree: QFileSystemModel + QTreeView (dirs only)
- File list: QListWidget showing supported files with sizes
- Quick preview: info label + PreviewTableView (first 100 rows)
- "Browse..." button to pick folder, or click tree to navigate
- "Open in Single File" button: emits signal -> switches to Single File tab, loads file
- "Set as Batch Input" button: emits signal -> switches to Batch tab, sets folder + config
- Double-click file in list = open in Single File tab

APP ARCHITECTURE (after Feature 6):
- MainWindow has QTabWidget with 3 tabs:
  1. FileBrowserTab - browse folders, preview files
  2. SingleFileTab - load, map, preview, export single file + save/load config
  3. BatchTab - batch process folder with saved config
- Cross-tab communication via signals:
  - FileBrowserTab.open_in_single -> MainWindow._open_file_in_single
  - FileBrowserTab.set_batch_input -> MainWindow._set_batch_folder
  - MainWindow passes current config from SingleFileTab to BatchTab
