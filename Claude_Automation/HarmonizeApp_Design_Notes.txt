PROJECT GOAL
------------
Build an interactive, standalone desktop application (.exe) for battery test
data harmonization. Replaces the script-based harmonize_run.py with a GUI
that non-technical battery test engineers can use.

USER REQUIREMENTS (from Q&A)
-----------------------------
- Framework: PySide6 (LGPL, free for commercial/selling)
- Scope: Battery-specific (16 unified target columns baked in)
- Transforms: Column mapping + formula builder + custom Python functions
- Config format: JSON files (not Excel)
- Pipeline scope: Harmonization + file browser (not archive extraction)
- Processing: Both single file and batch folder modes
- Location: battery_analysis_scripts/Claude_Automation/HarmonizeApp/

2. SUPPLIER DETECTION (hm_supplier_config.py)
----------------------------------------------
detect_supplier(filepath: Path) -> str
- Analyzes file path and name to identify supplier
- Returns: 'MCM', 'SRF', 'GOTION', 'TRURON', 'SZ', or 'UNKNOWN'
- Pattern examples:
  - "LFP44X" in path -> MCM
  - "096_" and "_P_" in path -> SZ
  - "DQ" prefix or "_DQ" -> GOTION
  - "CNSRF" in filename -> SRF
  - "CNTRURON" or "_Channel_" -> TRURON


3. CONFIG CLASSES (hm_import_data.py)
--------------------------------------
8 near-identical classes (cfg_mcm_std_01, cfg_mcm_std_02, cfg_mcm_exp_02,
cfg_mcm_xls_01, cfg_srf_std_01 x2, cfg_got_std_01, cfg_tru_std_01,
cfg_got_c32_01, cfg_sz_std_01)

Each has:
- __init__(filepath) - stores file path
- get_sheet_names() - returns Excel sheet names
- get_raw_data(config_selected) - reads file, cleans data, returns DataFrame

Key functions:
- find_matching_config(file_path, etl_df, hm_status_dict) -> (config_name, status)
  Matches file to correct config using supplier + filename pattern + sheet names

- run_harmonize_with_config(file_path, etl_df, hm_status_dict, config_name) -> (df, status)
  Executes harmonization: instantiate config -> load data -> apply transforms

- export_to_harmonized_folder(file_path, harmonized_data, harmonized_folder, copy_action) -> Path
  Exports to CSV with copy_action: 'skip_copy', 'create_copy', or 'replace'


4. DATA CLEANING (hm_general_support.py)
------------------------------------------
gen_clean_datasheet(sheet_df, config_selected, focus_cols, mandatory_cols):
  1. Auto-detect header row (uses Header_row_num from config, or searches for "step")
  2. Strip whitespace from column names
  3. Convert numeric string columns to numeric types
  4. Map columns via check_header_to_cfg_cols()
  5. Validate mandatory columns present
  Returns: (validation_flag, cleaned_df, header_mapping_df)

check_header_to_cfg_cols(sheet_df, cfg_selected, select_cols):
  Creates mapping matrix: config columns -> file columns
  Handles:
  - Simple column name: direct mapping
  - "col1 || col2": OR logic, pick first matching
  - "$$function_name(args)": Level 1 transformation
  - "$$$function_name(args)": Level 2 transformation (post-mapping)

  Returns DataFrame with columns:
  - Main_header: target column name
  - Cfg_header: config specification
  - File_header: actual file column or "function()"
  - Map_flg: boolean success


5. TRANSFORMATION SYNTAX (the $$ notation)
--------------------------------------------
From ETL config Excel and parsed by gen_extract_transform_fn():

Level 1 ($$): Applied to raw source data
  Syntax: $$function_name("col1", "col2")
  Regex: r"\$\$(\w+)\((.*)\)"

Level 2 ($$$): Applied to already-harmonized data
  Syntax: $$$function_name("col1")
  Applied after all level-1 transforms

OR syntax within args: "Col1 || Col2" -> use first existing column

Example ETL config row:
  Config_id = cfg_mcm_std_01
  Voltage_V = Voltage (V)                              <- direct map
  Current_A = $$mcm_transform_direction("I(A)", "State") <- level 1
  Power_W = $$$gen_calc_power()                         <- level 2


6. TRANSFORMATION APPLICATION (hm_gen_trans_func.py)
------------------------------------------------------
gen_apply_transform_raw_data(file_data, cfg_mapping, focus_columns):
  Phase 1: Direct column copy (simple mappings)
  Phase 2: Level 1 transforms ($$) on raw data via globals()[func_name]
  Phase 3: Level 2 transforms ($$$) on harmonized data via globals()[func_name]


7. ALL EXISTING TRANSFORM FUNCTIONS (~30 total)
-------------------------------------------------

MCM (hm_mcm_trans_func.py):
  parse_d_hms_ms(series) -> timedelta       # "d.hh:mm:ss.ms" parser
  mcm_transform_reltime(df, [col1, col2])   # Elapsed seconds from Test Time + Step Time
  mcm_transform_reltime_from_steptime(df, [col])  # From step time only
  mcm_transform_reltime_from_totaltime(df, [col]) # From total time only
  mcm_transform_unixtime(df, [date, test, step])  # Unix time with gap correction
  mcm_transform_Tcold(df, [T5, T6])        # Conditional: T5 if T6 exists
  mcm_transform_direction(df, [current, state]) # Negate on discharge
  mcm_rename_step(df, [step])              # R->Rest, C->Charge, D->Discharge, O->Control
  convert_mcm_xls_to_df(path)             # Legacy .xls reader

GOTION (hm_got_trans_func.py):
  got_rename_step(df, [step])              # CCCharge->Charge, CCDisCharge->Discharge, etc.
  got_rename_step_chinese(df, [step])      # Chinese: 静置->Rest, 充电CC-CV->Charge, 放电DC->Discharge
  got_transform_reltime(df, [time])        # time - time.iloc[0]
  got_transform_reltime_abs(df, [abstime]) # Relative seconds from first timestamp
  got_transform_unixtime(df, [abs, total]) # Gap-corrected timestamp
  got_transform_unixtime_abs(df, [abs])    # UTC datetime
  got_transform_Tcold(df, [T4, TempBox])   # Conditional: T4 if TempBox exists

SRF (hm_srf_trans_func.py):
  parse_d_hms_ms(series)                   # Same as MCM
  srf_transform_reltime(df, [test, step])  # Same logic as MCM
  srf_transform_reltime_from_steptime(df, [col])
  srf_transform_reltime_from_totaltime(df, [col])
  srf_transform_unixtime(df, [date, test]) # Gap-corrected
  srf_transform_direction(df, [current, state]) # Negate on discharge
  srf_rename_step(df, [step])              # Extended mapping including CCCV variants

TRURON (hm_tru_trans_func.py):
  tru_rename_step(df, [step])              # Same as GOT mapping
  tru_get_stepname(df, [step_idx, current]) # Infer from current median
  tru_transform_reltime(df, [time])        # time - time.iloc[0]
  tru_transform_reltime_abs(df, [abstime])
  tru_transform_unixtime(df, [abs, rel])   # Gap-corrected
  tru_transform_unixtime_abs(df, [abs])
  tru_get_step_capacity(df, [step, ch_cap, dch_cap]) # Per-step net capacity

SZ (hm_sz_trans_func.py):
  SZ_transform_capacity(df, [total, step]) # col1 - col2
  SZ_transform_energy(df, [total, step])   # col1 - col2
  SZ_name_step(df, [current])              # From current sign: <0 Discharge, >0 Charge, =0 Rest

Generic (hm_gen_trans_func.py):
  gen_calc_power(df)                       # Voltage_V * Current_A (level 2)
  gen_calc_unix(df)                        # Date_time.timestamp() (level 2)

All transform functions follow signature:
  func(df: pd.DataFrame, input_cols: list[str]) -> pd.Series | None


8. END-TO-END PIPELINE FLOW
-----------------------------
Raw supplier file
  -> detect_supplier() -> supplier ID
  -> find_matching_config() -> config class name
  -> Instantiate config class -> get_raw_data()
     -> Read Excel/CSV
     -> gen_clean_datasheet() -> fix headers, convert types, validate
     -> check_header_to_cfg_cols() -> column mapping matrix
  -> gen_apply_transform_raw_data()
     -> Phase 1: direct column copy
     -> Phase 2: level 1 transforms on raw data
     -> Phase 3: level 2 transforms on harmonized data
  -> Validation & cleanup
  -> export_to_harmonized_folder() -> CSV

CORE CLASSES
------------

MappingConfig (config_manager.py):
  name, description, version, file_settings, column_mappings
  Each ColumnMapping has mapping_type:
    'direct'           -> copy from one source column
    'or_fallback'      -> try columns in order, use first found
    'formula'          -> sandboxed expression (e.g., col_A * col_B)
    'builtin_transform'-> registered function by name + args
    'custom_function'  -> loaded .py file function
    'unmapped'         -> intentionally empty
  Level 1 = operates on raw source data
  Level 2 = operates on partially-built output, has depends_on field

Harmonizer (harmonizer.py):
  1. Execute all level-1 mappings
  2. Topological sort level-2 mappings by depends_on
  3. Execute level-2 mappings on partial output
  4. Return HarmonizeResult: output_df + validation + per-column status + errors

FormulaEngine (formula_engine.py):
  - Parse with ast module
  - Whitelist: +, -, *, /, **, abs, round, min, max, cumsum, diff, clip, fillna, .iloc, .shift
  - Block: import, exec, eval, open, __ attributes
  - Execute with restricted namespace

TransformRegistry (transform_registry.py):
  - _builtin: dict of ~30 existing functions
  - _custom: dict of user-loaded functions
  - All follow: func(df, input_cols) -> Series | None

FileLoader (file_loader.py):
  - load(filepath, sheet_name, header_row, encoding, separator) -> DataFrame
  - detect_header_row(df) -> int
  - get_sheet_names(filepath) -> list[str]
  - Adapted from gen_clean_datasheet() but decoupled from config-Excel

LEGACY TRANSFORM IMPORT STRATEGY
----------------------------------
adapters/legacy_transforms.py:
- Explicit imports (NO wildcard imports)
- Graceful fallback if legacy code unavailable (ImportError caught)
- Adds TB_CPA_TestDataAutomation parent to sys.path
- Registers all ~30 functions in TransformRegistry at startup

Note: Legacy transform files wildcard-import from dependencies.py which
pulls in patoolib, magic, winsound, etc. HarmonizeApp requirements.txt
must include python-magic-bin and tabulate to avoid import failures,
OR create thin adapter wrappers that mock unused imports.

For PyInstaller: Must add hiddenimports for all legacy transform modules
since they're dynamically imported.

KEY DESIGN DECISIONS
--------------------
1. Dropdown-based mapping (not drag-and-drop) - more accessible, handles
   OR/formula/transform cases naturally
2. AST-based formula sandboxing - whitelist validation then restricted eval
3. Keep existing transform signature: func(df, input_cols) -> Series | None
4. QThread for batch ops (not QProcess) - simpler, I/O-bound work
5. Self-contained JSON configs - no runtime dependency on Excel ETL config
6. Custom .py loading via importlib.util.spec_from_file_location
7. Explicit level field + depends_on for transform ordering (replaces $$/$$$ syntax)
8. PyInstaller --onedir mode for faster startup (not --onefile)
